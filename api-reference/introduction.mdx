The Olira REST API gives you programmatic access to every capability in the Olira platform. All requests are made over HTTPS to the base URL `https://api.olira.ai/v1`, and every response body is JSON. The API is versioned — the current version is `v1`, and the version identifier is part of the base URL so you always know exactly which contract your code is running against.

## Base URL

Every API endpoint is relative to the following base URL:

```
https://api.olira.ai/v1
```

## Sandbox

Before going live, you can develop and test against the Olira sandbox environment:

```
https://api-sandbox.olira.ai/v1
```

The sandbox is an isolated environment that mirrors the production API surface but never affects real data. To use it, generate a test API key from the **Sandbox** section of your dashboard at [https://app.olira.ai](https://app.olira.ai) and point your requests at the sandbox base URL.

<Note>
  Sandbox API keys are prefixed with `sk_test_` and will not work against the production base URL. Likewise, live keys prefixed with `sk_live_` are rejected by the sandbox.
</Note>

## Versioning

The current stable API version is **`v1`**. The version is embedded in the base URL, so your integration is always pinned to a specific contract.

Olira follows these versioning rules:

- **Breaking changes** — for example, removing a field or changing a field's type — will increment the version to `v2`. Olira will give advance notice before a new version becomes the default, giving you time to migrate.
- **Backwards-compatible changes** — for example, adding a new optional field or a new endpoint — are rolled out to the current version without a version bump.

If you depend on strict response shapes, we recommend validating only the fields your code actually uses so that additive changes do not break your integration.
