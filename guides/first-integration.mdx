This guide walks you through the complete lifecycle of an Olira resource — from creation to deletion. By the end, you will have made authenticated requests to create, retrieve, update, list, and delete a resource using the Olira REST API at `https://api.olira.ai/v1`. All requests require a Bearer token in the `Authorization` header, so make sure you have your API key ready before you begin.

<Steps>
  <Step title="Create a resource">
    Send a `POST` request to `/resources` with a JSON body containing the resource name. The API responds with the newly created resource object, including its unique `id` — keep this handy for the steps that follow.

    <CodeGroup>
      ```bash cURL
      curl --request POST \
        --url https://api.olira.ai/v1/resources \
        --header 'Authorization: Bearer <API_KEY>' \
        --header 'Content-Type: application/json' \
        --data '{
          "name": "My First Resource"
        }'
      ```

      ```javascript JavaScript
      const response = await fetch("https://api.olira.ai/v1/resources", {
        method: "POST",
        headers: {
          "Authorization": "Bearer <API_KEY>",
          "Content-Type": "application/json",
        },
        body: JSON.stringify({ name: "My First Resource" }),
      });

      const resource = await response.json();
      console.log(resource);
      ```
    </CodeGroup>

    **Response**

    ```json
    {
      "data": {
        "id": "res_01HXYZ",
        "name": "My First Resource",
        "created_at": "2024-01-15T10:30:00Z",
        "updated_at": "2024-01-15T10:30:00Z"
      }
    }
    ```
  </Step>

  <Step title="Retrieve the resource">
    Use the `id` returned in the previous step to fetch the resource directly with a `GET` request to `/resources/{id}`.

    <CodeGroup>
      ```bash cURL
      curl --request GET \
        --url https://api.olira.ai/v1/resources/res_01HXYZ \
        --header 'Authorization: Bearer <API_KEY>'
      ```

      ```javascript JavaScript
      const response = await fetch("https://api.olira.ai/v1/resources/res_01HXYZ", {
        method: "GET",
        headers: {
          "Authorization": "Bearer <API_KEY>",
        },
      });

      const resource = await response.json();
      console.log(resource);
      ```
    </CodeGroup>

    **Response**

    ```json
    {
      "data": {
        "id": "res_01HXYZ",
        "name": "My First Resource",
        "created_at": "2024-01-15T10:30:00Z",
        "updated_at": "2024-01-15T10:30:00Z"
      }
    }
    ```
  </Step>

  <Step title="Update the resource">
    Send a `PATCH` request to `/resources/{id}` with only the fields you want to change. The API returns the full updated resource object.

    <CodeGroup>
      ```bash cURL
      curl --request PATCH \
        --url https://api.olira.ai/v1/resources/res_01HXYZ \
        --header 'Authorization: Bearer <API_KEY>' \
        --header 'Content-Type: application/json' \
        --data '{
          "name": "Updated Name"
        }'
      ```

      ```javascript JavaScript
      const response = await fetch("https://api.olira.ai/v1/resources/res_01HXYZ", {
        method: "PATCH",
        headers: {
          "Authorization": "Bearer <API_KEY>",
          "Content-Type": "application/json",
        },
        body: JSON.stringify({ name: "Updated Name" }),
      });

      const updated = await response.json();
      console.log(updated);
      ```
    </CodeGroup>

    **Response**

    ```json
    {
      "data": {
        "id": "res_01HXYZ",
        "name": "Updated Name",
        "created_at": "2024-01-15T10:30:00Z",
        "updated_at": "2024-01-15T10:35:00Z"
      }
    }
    ```
  </Step>

  <Step title="List all resources">
    Retrieve a paginated list of all resources in your account by calling `GET /resources`. The response includes a `meta` object with pagination details. See the [Pagination guide](/guides/pagination) for techniques on iterating through large collections.

    <CodeGroup>
      ```bash cURL
      curl --request GET \
        --url 'https://api.olira.ai/v1/resources?page=1&per_page=20' \
        --header 'Authorization: Bearer <API_KEY>'
      ```

      ```javascript JavaScript
      const response = await fetch(
        "https://api.olira.ai/v1/resources?page=1&per_page=20",
        {
          method: "GET",
          headers: {
            "Authorization": "Bearer <API_KEY>",
          },
        }
      );

      const result = await response.json();
      console.log(result.data);       // array of resources
      console.log(result.meta.total); // total count across all pages
      ```
    </CodeGroup>

    **Response**

    ```json
    {
      "data": [
        {
          "id": "res_01HXYZ",
          "name": "Updated Name",
          "created_at": "2024-01-15T10:30:00Z",
          "updated_at": "2024-01-15T10:35:00Z"
        }
      ],
      "meta": {
        "total": 1,
        "page": 1,
        "per_page": 20
      }
    }
    ```
  </Step>

  <Step title="Delete the resource">
    Send a `DELETE` request to `/resources/{id}` to permanently remove the resource. A successful deletion returns an HTTP `204 No Content` response with an empty body.

    <CodeGroup>
      ```bash cURL
      curl --request DELETE \
        --url https://api.olira.ai/v1/resources/res_01HXYZ \
        --header 'Authorization: Bearer <API_KEY>'
      ```

      ```javascript JavaScript
      const response = await fetch("https://api.olira.ai/v1/resources/res_01HXYZ", {
        method: "DELETE",
        headers: {
          "Authorization": "Bearer <API_KEY>",
        },
      });

      if (response.status === 204) {
        console.log("Resource deleted successfully.");
      }
      ```
    </CodeGroup>

    **Response**

    ```
    HTTP 204 No Content
    ```
  </Step>
</Steps>

<Tip>
  Now that you have mastered the CRUD lifecycle, set up [webhooks](/guides/webhooks) to receive real-time notifications whenever a resource is created, updated, or deleted — no polling required.
</Tip>
