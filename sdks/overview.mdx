Olira's official SDKs give you a productive, idiomatic way to integrate with the Olira platform without writing raw HTTP requests. Each SDK wraps the REST API (base URL `https://api.olira.ai/v1`) with fully typed methods, automatic retries, pagination helpers, and structured error objects — so you can focus on building your product instead of managing infrastructure.

## Why use the SDK?

You can always call the Olira REST API directly, but the SDKs offer several advantages that make day-to-day development faster and more reliable:

- **Type safety** — every request parameter and response field is fully typed, giving you autocomplete and compile-time checks in TypeScript and type-annotated signatures in Python.
- **Automatic retry logic** — transient network errors and rate-limit responses are retried with exponential back-off, so you don't have to implement that logic yourself.
- **Built-in pagination helpers** — iterate over large result sets without manually managing page cursors or constructing successive requests.
- **Clear error objects** — errors surface as structured `OliraError` instances with a machine-readable `code` and a human-readable `message`, making error handling consistent across your codebase.
